## 问题1: 字母键Q/W/E/A/S/D/Z/X与MuJoCo冲突

### 实测发现的冲突
- **Q**: 显示camera位置
- **W**: 显示网格线/纹理
- **A**: 显示骨架
- **S**: 显示阴影
- **D**: 切换背景黑/白

### 解决方案
改用数字键**2-9**（跳过0和1，0/1仍有MuJoCo内部绑定）

---

## 问题2: 状态行显示被截断

### 现象
```
| base_yaw D 0.1[100%] | body_pitch N | [MANUAL] yaw=D(1.5s) pitch=N(0.3s
                                                              ^^^^ 被截断
```

### 原因
清除缓冲区长度不足（120字符 < 实际输出长度）

### 解决方案
- 扩展清除长度：120 → **200字符**
- 重新设计状态行为固定宽度格式

## 问题3: 状态行格式不对齐

### 新格式设计

**固定宽度对齐**，手动挡计时器嵌入到每个轴内：

```
[SIM] replaying | frame 2127/3036 | 31/s || D  11.7 [  7%] ( 1.5s) | D   0.1 [100%]        | N        ||
                                             ↑   ↑     ↑     ↑        ↑   ↑     ↑            ↑
                                          档位 阻尼   百分比 手动倒计时   档位 阻尼  百分比    自动挡(空格占位)
```

### 格式规范

每个轴的显示：
```
{gear} {damping:5.1f} [{pct:3.0f}%] ({manual:3.1f}s)
 ↑         ↑             ↑               ↑
 1字符   5字符+1空格   5字符        7字符（或7空格）

总宽度：1 + 1 + 5 + 1 + 5 + 1 + 7 = 21字符（固定）
```

- **自动挡**: 手动计时器位置填7个空格
- **手动挡**: 显示倒计时 `(1.5s)`

### body_pitch特殊处理
躯干俯仰无D档，只显示：
```
{gear} ({manual:3.1f}s)
  或
{gear}        
```

## 最终键位方案

```
2  3  4     base_trans  (底盘平移):  P / N / D
5  6  7     base_yaw    (底盘旋转):  P / N / D
8  9        body_pitch  (躯干俯仰):  P / N
```

### 为什么不用0和1？
- **0**: MuJoCo内部用于相机切换
- **1**: MuJoCo内部切换geom category 1可见性（导致机器人消失）

### 记忆方法
- **234**（前段）: 平移
- **567**（中段）: 旋转
- **89**（后段）: 俯仰
