# 全身遥操作的意图驱动模式切换报告

> 对象:`avp_teleop_upper_body/`（Astribot S1，23-DOF 合并 Pinocchio+Pink 差分 IK，60 Hz）
> 主题:把操作者的运动意图（头 + 双手 6-DoF）连续、平滑地映射到机器人冗余自由度的调度机制
> 类比:汽车自动变速箱 TCU —— 用多个意图信号在一张"换挡图"上决定每个自由度的"档位"

---

## 1. 问题背景

系统用一个合并的差分 IK（QP）联合求解 23 个自由度（移动底盘 x/y/yaw + 躯干 4 + 颈 2 + 双臂 14），跟踪头相机 + 左右手三个末端目标。冗余（23 DOF 跟踪 ≤18 维任务）由**逐自由度的速度阻尼代价**解析:某个关节阻尼越高，QP 越"不愿意"动它，越晚被征用。

**核心矛盾**:同一套固定的阻尼配置无法同时满足所有场景。
- 操作者**静止精细操作**时:希望底盘/躯干**绝对不动**（否则 AVP 抖动会让底盘乱转、膝盖发抖）。
- 操作者**大范围走动/转身/下蹲**时:希望底盘/躯干**主动参与**（否则手臂扭曲去够远处目标）。

固定阻尼要么"太活"（静止抖动）要么"太钝"（移动时手臂扭曲）。**需要让控制行为随操作者意图动态变化。**

---

## 2. 核心思想:三个正交的模式切换维度

把机器人的自由度按**运动模式**解耦成三个**正交的轴**,每个轴用一个**独立的意图信号**门控,互不干扰。三个轴共享同一套底层机制,但各自独立决定"档位"。

| 维度 | 控制的自由度 | 意图信号 | 物理含义 |
|---|---|---|---|
| **① 平移 base_trans** | 底盘 x, y | 头**水平**速度 (m/s) | 操作者是否在走动 |
| **② 旋转 base_yaw** | 底盘 yaw | **组合偏航率**(rad/s) | 操作者是否在转身/转腰 |
| **③ 下蹲 body_pitch** | 前倾脊柱 torso 1/2/3 | 头**垂直**速度 (m/s) | 操作者是否在下蹲/弯腰 |

**为什么正交很关键**:三个信号相互独立,所以机器人能同时处于不同档位。例如"边走边转身"时,平移和旋转两个轴各自解禁;而"原地扭腰"时,平移保持锁定、只有旋转解禁——这正是早期版本失败的地方(见 §5)。

### 组合偏航率(旋转轴的信号,一个创新点)
旋转意图**不能只看头的朝向**:操作者"原地转腰"时头不转、只有双手和躯干在扫。因此旋转信号取两项之和:

```
组合偏航率 = 头偏航率 + (双手中点→头)连线的偏航率
```

- 只有双手扫动（转腰，头不动）→ 组合信号高 → 底盘接管转向。
- 只有头转（转头看）→ 组合信号高。
- 两者都转（大转身）→ 组合信号更高 → 底盘全力接管。

（头偏航率取自头部**耳间轴**的水平投影,实测该轴在低头/深蹲时仍保持水平,不像视线轴会退化。）

## 3. 档位:P / N / D 的连续过程

每个维度沿其意图信号轴划分为**三个连续档位**（借鉴自动挡的 P/N/D）:

```
信号强度  0 ────── P ──────阈值1───── N ──────阈值2───── D ──────►
                 (冻结)         (解禁,静态阻尼)     (解禁,阻尼连续下降)
```

| 档位 | 状态 | 机制 |
|---|---|---|
| **P (Park)** | 冻结 | 该自由度速度上限设为 **0** —— QP 内部硬约束,其余关节协调补偿(coordinated freeze) |
| **N (Neutral)** | 解禁,静态阻尼 | 速度上限恢复,阻尼 = 静态值(过渡区:已解禁但意图还不够强) |
| **D (Drive)** | 解禁,连续调度 | 阻尼随意图信号**连续下降**到地板值,该自由度越来越"主动" |

**关键设计**:
- **P→N 边界**用**迟滞双阈值**(freeze / unfreeze),防止信号在边界附近抖动式反复切换。
- **N→D 区间**阻尼是意图信号的**连续函数**(线性插值 + EMA 平滑),避免"换挡冲击"(阻尼突变导致 QP 解跳变)。
- **底层机制统一**:P 档就是"阻尼→∞"的极限(直接零速度上限);D 档是连续调阻尼。二者是同一根轴的两端。

### 各维度的档位边界与阻尼范围(当前默认)

| 维度 | P→N 阈值 | N→D 阈值 | D 区阻尼范围 |
|---|---|---|---|
| **① 平移** | 头水平速度 0.05 / 0.12 m/s | 0.20 m/s | 12 → 8（gentle）|
| **② 旋转** | 组合偏航率 0.15 / 0.40 rad/s | 0.50 rad/s | 8 → 2（激进）|
| **③ 下蹲** | 头垂直速度 0.08 / 0.20 m/s | （无 D 档，见下）| — |

> **body_pitch 只有 P/N**:前倾脊柱的运动由平衡任务(保持胸在踝上方)和死区控制,阻尼只是低优先级正则,没有连续调度的 D 档。这是刻意的——它不需要"越来越主动"的语义。

### 状态行实时显示
运行时终端每 2 秒刷新三维档位,直观可读:
```
[SIM] replaying | frame 512/1200 | 60/s | base_trans D 9.5[62%] | base_yaw D 5.0[50%] | body_pitch N
```
- `P` = 冻结;`N 12` = 解禁静态阻尼 12;`D 9.5[62%]` = 调度中,实时阻尼 9.5,解禁程度 62%(0%=刚进 D,100%=到地板)。

## 4. 实现要点

### 底层杠杆:两种,都在 QP 内部生效
1. **冻结(P 档)**:把某组自由度的速度上限(`model.velocityLimit`)设为 0。Pink 每次求解实时读取,所以逐帧切换立即生效、无需重建;且是**协调冻结**——QP 让其余关节绕着被钉住的自由度重新求解,而非事后截断。
2. **连续调阻尼(D 档)**:实时修改 `DampingTask` 的逐自由度代价向量。同样每次求解实时读取。**已验证**:底盘偏航阻尼 8→200 可让一个远目标的转角从 1.22 rad 降到 ≈0。

两个杠杆是同一根轴的两端(阻尼→∞ 即冻结),所以 P/N/D 是一个连续的机制族。

### 与"死区"的关系:保留,不推翻
业界文献倾向"完全放弃死区、全程连续调度"。**我们保留了死区**,理由基于实测:AVP 噪声在**零速度附近**最大,死区在那个区间硬冻结、最干净;连续调度只在"速度已明显高于噪声"的区间(N/D)介入。二者**优势互补**——死区管零速噪声,调度管动态意图。这个"先死区、动起来再连续换挡"的分层规避了纯连续方案在零速端的抖动风险。

### 平滑保证
- 阻尼本身经 EMA 平滑(不是瞬间跳变),避免换挡冲击。
- 迟滞双阈值防止 P↔N 边界抖动。
- 单调映射 + 下方死区保证驻留时间,规避切换系统的稳定性隐患。

## 5. 取得的成果

### 成果一:静止抖动根治(平移 P 档 + 下蹲 P 档)
| 场景 | 指标 | 关闭 → 开启 |
|---|---|---|
| 静止精细操作(clip1-3) | 底盘累计运动 | 0.115 → **0.003**(−97%),冻结 100% |
| 静止精细操作(clip1-3) | 前倾脊柱累计运动 | 1.52/0.63/0.33 → **0.00**（膝盖抖动消除）|

**手部跟踪零损失**(0.1mm 不变)——静止时底盘和膝盖完全不动,精细操作稳定。

### 成果二:平移/旋转解耦(正交轴的核心价值)
早期版本用单一开关(平移速度)冻结整个底盘,导致**原地转身时 yaw 被平移连带锁死** → 转身失败(手臂扭曲)。拆成正交轴后:

| 场景 | 平移 xy 冻结% | 旋转 yaw 冻结% | 效果 |
|---|---|---|---|
| **直线走**(clip9) | 26%（解禁走路）| **82%（保持冻结）** | 走路时底盘不乱转 |
| **原地大转腰**(clip7) | 30%（多数冻结）| **6%（解禁转向）** | 底盘接管转向(转 14.45 rad),手臂不扭 |
| **360°转身**(clip11) | 26% | **51%（按需解禁）** | 转身时底盘接管——早期版本的失败点被根治 |
| **静止**(clip1) | 100% | 92% | 两轴都冻结 |

自检 `check_base_axis_split` 验证正交性:仅冻 xy → 底盘只转不走(1.47 rad / 0 cm);仅冻 yaw → 只走不转(48.7 cm / 0 rad)。

### 成果三:平移连续调度不牺牲精度(D 档 gentle 验证)
| 场景 | 底盘位移 关→开 | 手部误差 关→开 |
|---|---|---|
| 前后走(clip9) | 9.28 → 9.47 m | 8.7 → **7.0 mm** |
| 左右走(clip10) | 7.25 → 7.37 m | 48.1 → **45.6 mm** |

走路时底盘略微更主动,手部误差**不降反而略好**（底盘多帮一点,手臂少伸一点)。

### 成果四:统一、可扩展、可回退的架构
- 三个正交轴共享同一套机制(冻结 + 调阻尼),各配一个意图信号。
- **每个功能默认可关**、可 A/B、逐轴独立调参(CLI 暴露全部阈值)。零回归:关掉调度即退回上一版行为。
- 未来新增维度(如颈部)可用同样方式接入。
- **离线自检 22/22 全通过**,覆盖每个维度的冻结/解禁/调度正确性。

## 6. 方法学

- **可复现评估**:先建输入录制/回放基础设施(戴一次 AVP 录下原始输入,之后离线反复回放),11 段录制轨迹覆盖静止/走动/转身/下蹲各场景。
- **测量用 IK-only**:回放输入→跑完整重定向+IK→前向运动学算跟踪误差,不走物理步进(物理伺服在剧烈动作上会发散,是独立稳定性问题,会污染度量)。
- **数据驱动的默认值**:每个阈值来自实测的意图信号分布——例如深蹲垂直速度 p90≈1.25 m/s vs 静止 p90≈0.01(分离度 ~100×),据此定下蹲阈值 0.08/0.20。
- **measure-then-default**:每个新功能先默认关、跑数据、确认不变差(或明确改善)才考虑设为默认。

## 7. 命令行调参

三维全开:
```bash
mjpython -m avp_teleop_upper_body.sim_teleop --orientation --init-pose my_pose1 \
    --replay-avp clip11 --enable-trans-scheduling --enable-yaw-scheduling
```

| 维度 | 开关 / 阈值 flag | D 档地板 flag |
|---|---|---|
| ① 平移 | `--enable-trans-scheduling`、`--base-freeze/unfreeze-speed`、`--trans-schedule-speed-low/high` | `--trans-schedule-floor` |
| ② 旋转 | `--enable-yaw-scheduling`、`--base-yaw-freeze/unfreeze-rate`、`--yaw-schedule-rate-low/high` | `--yaw-schedule-floor` |
| ③ 下蹲 | `--lean-freeze/unfreeze-speed`（死区常开）| —（无 D 档）|

调参直觉:降 unfreeze 阈值 → 更早解禁;降 D 档地板 → 解禁后更主动;降 EMA alpha → 换挡更柔和。
