# AVP 全身遥操作稳定性改进报告

> 对象:`avp_teleop_upper_body/`(Astribot S1 全身遥操作,23-DOF 合并 Pinocchio+Pink IK)
> 方法:11 段真机录制轨迹 → 问题诊断(4 根因)→ 分阶段修复(Phase 0–3)→ 根因定位
> 结论:2 个阶段落地生效;3 个阶段经数据证伪、默认关闭,但三次独立否定共同锁定了真正的根因

---

## 1. 背景

系统把操作者的 Apple Vision Pro **头 + 双手** 6-DoF 位姿,通过一个合并的差分 IK(QP)映射到 S1 的 **23 个自由度**:移动底盘(x/y/yaw)+ 躯干(4)+ 颈(2)+ 双臂(14)。三个末端任务(头相机、左工具、右工具)在同一个 `pink.Configuration` 里联合求解,手臂自动补偿躯干、底盘按需延伸可达。

为可复现评估,先建了**输入录制/回放**基础设施:戴一次 AVP 录下原始输入(存 `avp_trajectory/`),之后离线反复回放跑完整重定向,无需再戴头套。11 段实验即基于此。

---

## 2. 实验:11 段录制轨迹

按运动类型设计了 11 段任务,覆盖精细操作、大跨度、高动态、原地转身、平移、整体转身。观察到的问题:

| 轨迹 | 任务 | 观察到的现象 |
|---|---|---|
| json1–3 | 纯末端精细控制(仅小幅动手腕/小臂,身体僵直) | 底盘偶尔**原地旋转**、膝盖偶尔**抖动** |
| json4 | 中小幅度动小臂/肘/肩 | 偶尔**全身剧烈抖动** |
| json5 | 大跨度、跨中轴线、接近手臂伸直极限 | 偶尔**全身剧烈抖动** |
| json6 | Z 轴高动态(深蹲拾物、快速拳击后仰) | 基本正常,弯腰时**底盘前后些许漂移** |
| json7,8 | 原地偏航扭转(脚不动,靠腰腹扭身回头) | 大幅旋转时底盘/腰**都不动、只动手强行够 → 手臂扭曲**;json8 出现 **NaN 报错**(`QACC at DOF 3`,仿真发散) |
| json9,10 | 平移(前后走 3m / 左右走 3m) | json9 后退时**手一直在后面追、拖着机器人走**;json10 **上肢扭曲失败** |
| json11 | 前进 + 360° 转身 + 返回 | **失败**,双臂**交叉扭曲于身后** |

---

## 3. 诊断:4 个根因

把 11 段现象归纳为 4 个根因:

| 根因 | 症状 | 对应轨迹 |
|---|---|---|
| **A. 没有"移动/转身意图 → 底盘"的显式通道** | 大范围移动/转身时底盘和腰不动,手臂扭曲去够 | json7,8,9,10,11 |
| **B. 离群包/近奇异 → 目标跳变 → NaN/剧烈抖动** | 偶发全身剧烈抖动、NaN 崩溃 | json4,5,8 |
| **C. 静止时底盘太"积极"** | 底盘原地转、膝盖抖、手带动底盘颤 | json1–3、小范围移动 |
| **D. 手可跑到骨盆后方** | 手在身后追、双臂交叉扭曲 | json9,10,11 |

**核心洞察**:底盘原本只在"手够不到、frame 误差顶过阻尼"时**被动**移动(反应式)。没有任何信号告诉它"人在走/在转,该主动移动"——这是根因 A,也是 json7–11 全崩的根。

---

## 4. 分阶段修复

按 **风险低→高、收益即时→系统** 排序,每阶段都用 11 段 clip 离线回放做回归验证。评估用 **IK-only**(solve 后用 FK 算跟踪误差,不走物理步进)——因为发现 MuJoCo `mj_step` 位置伺服在剧烈 clip 上会发散(见 §6 方法学)。

### Phase 0 — 稳定性护栏(落地生效,治 B)

**两道独立防线,不改变正常输入下的行为:**
1. **求解器输出守卫**:QP 偶尔返回非有限/超大速度而不抛异常 → 积分后 NaN 流入 MuJoCo。守卫:`v` 非有限则置零;否则钳到速度上限;积分后 `q` 非有限则保持上一帧。**杜绝 NaN 到达仿真**。
2. **目标离群钳制**:限制目标每帧最大跳变(默认 0.08 m/帧 ≈ 4.8 m/s),压住坏包/丢包后大跳变;保留原 EMA。

**验证**:selfcheck 新增 2 项;clip8/5/4 全路径回放 **0 次 NaN**——json8 崩溃消失,clip4/5 抖动消除。

### Phase 1 — 底盘 dead-zone(落地生效,治 C)

头部水平不动时**在 QP 内部冻结底盘**(chassis 速度上限设 0,手臂协调补偿);头水平移动超阈值才解冻。用**迟滞双阈值**(0.05 冻结 / 0.12 解冻)防抖动式反复切换;只用**水平**头速门控,纯下蹲(json6)不误触发。

**验证**:
| clip | 场景 | 底盘累计运动 关→开 | 冻结时间占比 |
|---|---|---|---|
| clip1–3 | 静止 | 0.115→**0.003**,0.025→0,0.016→0 | **100%** |
| clip9 | 前后走 3m | 7.23→6.68(基本保留) | 26%(走路时解冻) |

静止乱转/手带底盘颤**根治**,移动能力不受影响。

### Phase 2 — 底盘跟随头部平移(证伪,默认关闭,针对 A)

尝试用头部水平位移主动驱动底盘。**机制建成并测试**(`BaseTrackingTask`,与冻结兼容,selfcheck 通过),但**策略被数据推翻**:

| clip | 底盘位移 关→开 | 手部误差 关→开 |
|---|---|---|
| clip9 | 0.301→0.144 m | **96→254 mm(更差)** |
| clip10 | 0.069→0.036 m | **144→510 mm(更差)** |
| clip11 | 0.718→0.217 m | **305→607 mm(更差)** |

**原因**:反应式底盘**已经**在最优地移动去服务手部目标了。头位移算出的底盘目标和手臂需要的不一致,反而和手部任务(cost 10)抢底盘自由度。**"用头位移驱动底盘"对平移是错的——手部目标本身已编码走路意图。**

### Phase 2b — 底盘/腰偏航跟随(证伪,默认关闭,针对 A 的转身部分)

Phase 2 让我们意识到:**平移反应式已够,真正缺的是转身信号**。先做腰 yaw 跟随(治 json7/8 原地扭腰)。

**关键副产物——找到 pitch-robust 转向信号**:实测头部各轴,发现我 Phase 2 误用的"前向"(头局部 +Z)其实**几乎竖直**(mean\|z\|=0.98),低头时投影退化 → 乱转。而**头部耳间轴(左右轴)全程水平**(深蹲 clip6 仍 mean\|z\|=0.03、投影≥0.99),是稳健的转向信号。

**机制建成并测试**(`WaistYawTask`,与 EgoPoser 互斥,selfcheck 通过),但**再次被推翻**:clip7 手臂偏离 10.62→11.06(更差),clip8 手部误差 7.6→13.9mm(更差)。**原因**:反应式的手部任务**已经把腰驱动到硬限位 ±1.53 了**,头yaw目标没有余量,只会竞争。

### Phase 3 — 手在骨盆前方软护栏(证伪,默认关闭,针对 D)

单侧软守卫:手越过骨盆后方平面才把它往前推,手在前方时**完全惰性**(不伤常见情况)。**机制建成并测试**(`HandFrontTask`,selfcheck 通过),但**第三次被推翻**:
- 对 **交叉(clip10/11)完全无效**(cross% 36→36)——交叉是**横向**问题,守卫是**前后**惩罚,轴不对。
- 对**手在后方(clip7/9)**只微改善却**伤跟踪**(clip7 24→35mm、clip9 349→440mm)。

---

## 5. 根因定位(三次否定的共同结论)

Phase 2、2b、3 **三次独立尝试全部失败,原因完全相同**——这本身是本工作最重要的结论:

> 反应式 23-DOF QP **正确跟踪了手部目标**;"手在身后/交叉"的坏姿态,是因为**手部目标本身就在身后/交叉**。手目标锚定在**固定世界点**(相对标定时刻:`target = tool0 + align_R·(wrist − wrist0)`)。当操作者走动/转身而底盘不跟随时,这些世界固定的目标就落到了机器人的身后/交叉位置。

**目标编码了失败。** 任何加在机器人构型上的任务(底盘跟随 / 腰跟随 / 手守卫)都**无法修一个本质就在身后的目标**——只能与它竞争、抢自由度、伤跟踪。三次否定从三个不同角度验证了同一个结论。

**真正的解法是架构级的:body-relative(身体相对)手部目标。** 把手目标锚定到一个**随身体移动的躯干/骨盆帧**(手本来就是相对身体的),而非固定世界点。这样操作者+底盘一起走/转时,手目标跟着身体走,**永远不会落到身后**。这是**重定向参考系的重构**(改 `wrist_to_tool_target` + 标定),不是再加一个 QP 任务。

---

## 6. 方法学说明(给导师看数据来源)

**评估不是"对轨迹积分",而是离线回放 + 两种测量:**

- **IK-only 评估(可信,报告主要数据来源)**:回放录制的 AVP 输入 → 走完整重定向 + IK 求解 → 用 `frame_pose` 前向运动学算"实际 vs 期望"跟踪误差。**不走物理步进**,无发散。这是本报告表格里的数字。
- **sim 评估(mj_step)一度误导**:早期用带物理步进的仿真评估,在剧烈 clip(clip7/8)上**位置伺服发散**,给出物理上不可能的数字(2000mm 手部误差、tick 3 就 NaN)。切换到 IK-only 后发现:**IK 其实把 clip7/8 的手跟踪到 6–45mm、从不 NaN**。所以渲染里看到的"手臂扭曲/崩溃",实际是 **(1) mj_step 仿真伺服发散**(独立稳定性问题)+ **(2) 合法的 IK 把腰用到极限**,不是 IK 跟踪失败。
- 一个 harness 坑:在同一进程循环里跑"关/开"对比,会把上一次发散的仿真状态泄漏到下一次;每次对比须用**独立进程**。

**离线自检**:全程维护 selfcheck,从初始 11 项增长到 **18/18 全通过**,覆盖每个新增机制(护栏、dead-zone、三个跟随/守卫任务的正确性与惰性、EgoPoser 互斥等)。

---

## 7. 结论与建议

**落地生效(默认开启,零回归):**
- Phase 0 稳定性护栏——消除 NaN 崩溃(json8)与剧烈抖动(json4/5)。
- Phase 1 底盘 dead-zone——根治静止时底盘乱转/手带底盘颤(json1–3)。

**证伪但保留(默认关闭,机制经测试,是未来 body-relative 重定向的现成积木):**
- Phase 2 底盘平移跟随、Phase 2b 腰/底盘偏航跟随、Phase 3 手在前守卫。

**科研价值**:三次独立否定共同锁定了根因——**症状守卫无效,问题在重定向的固定世界系参考。** 附带产出:pitch-robust 转向信号(头耳间轴)、可复现的输入录制/回放 + IK-only 评估基础设施。

**建议的下一步(按优先级):**
1. **Body-relative 手部目标重构**(治根)——把手目标锚到移动的躯干帧;
2. 排查 **clip7/8 的 mj_step 仿真伺服发散**(独立稳定性问题,可能是实机/渲染看到"崩溃"的真凶);
3. 若未来需要长程移动,平移跟随可做成**可选**模式,但评价指标应是"手臂姿态健康度(离奇异/极限的裕度)"而非末端误差。
